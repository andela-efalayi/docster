    Introduction

    Welcome to the Docster API. With this API you can get information on how to use Docster document management system [including creating, reading, updating and deleting roles, users, and dcouments.]

    You can view code examples in the dark area to the right.

    This API documentation page was created with Slate. Feel free to edit it and use it as a base for your own API's documentation.

    Roles

    Create Role

    Request:

    {
  "roleType": "facilitator",
}

    Response:

    {
    "role": {
        "id": 5,
        "roleType": "facilitator",
        "updatedAt": "2017-07-28T12:00:21.271Z",
        "createdAt": "2017-07-28T12:00:21.271Z"
    },
    "message": "Role created"
}

    Request

    • Endpoint: POST: /api/v1/roles
    • Requires: Authentication
    • Body (application/json)

    Response

    • Status: 201: Created
    • Body (application/json)

    Errors

    This section lists error status codes used in the app.

    The Docster API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request sucks.
    401 Unauthorized -- You have no token.
    403 Forbidden -- You are not an admin.
    404 Not Found -- User, role, or document cannot be found.
    500 Internal Server Error -- We had a problem with our server. Try again later.
